Hosting a Home Media Server - part 2

February 27, 2026 4-minute read
Videos • Vlog
Videos • Self-Host
In previous post, I wrote about my home server I use to host videos. I had some free time after quiting my job so I decided to finish it.

So in my previous post, the Jellyfin server is up and serving videos. However the quality is not as good as I expected. Turnout, to play 4K videos on Chrome, I have to enable hardware accelerated option in Chrome settings and also install the gpu driver as well. I have intel and nvidia gpus so I installed both. After that, the video is served as original 4K. Speed is now only depended on network. My PC doesn’t have to work too much to convert videos to different codec.

At this point, the server is only accessible by private IP on my home network. But I don’t want to use it for myself only. I want my wife to be able to access it as well. This requires my server be visible to the public internet.

So, my next step is to publish the server to the internet. For this task, I use no-ip as DDNS service provider. With this, I don’t have to buy a static public IP from ISP. Instead, anytime my home public IP is changed, no-ip will update DNS record to the later. It is free for one domain, and quite cheap if you want to add some more domains. You can use the code REFER20 to get 20% off any new service.

Public server with No IP Link to heading

In this section, I will go into details about setting up No IP. I made different configurations, try and update before the final version because this also relates ISP router and domain name.

Create a no-ip hostname Link to heading

The guide is quite easy to follow. Here I just note some important info.

In side no-ip account dashboard you can create hostname.

Type: must be A to map with your public IP
Host: the domain name you want, must be unique.
IPv4: no-ip will detect & fill current public IP visitting the site
Enable Dynamic DNS: yes, we need this

After that, you will see a dialog about DDNS key credentials. Save it somewhere safe or write it down for the next steps.

Access local website from the internet Link to heading

Now we want to access our local website from created no-ip domain. I used a simple “Hello world!” index.html file and serve it with python using python -m http.server. This will serve a simple web server on port 8000.

Now if you open browser and go to http://localhost:8000 you can see the “Hello world!” message. We want to visit http://myddns.ddns.net and see the same result. If you do this now, you will see only an error message.

This is done by setting up port forwarding on your router.

You can check out instructions for your router in the official guide. My router is provided by the ISP, so it wasn’t on the list, but the principal is the same. We want to configure our router to pass requests on matching ports to our host private IP.

First you need to login into admin account of the router. Find the “Port Forwarding” feature. For my Viettel home router, I can find it under Advanced settings > Internet > Security > Port Forwarding. The most important fields are LAN host, WAN Port and LAN Host Port.

  • LAN Host: this is the IP of my PC (192.168.1.162), you would want to set static IP on it as well.
  • WAN Port (or External Port): this is the port receive requests from the internet. We can set it to 80 since we want to see our website.
  • LAN Host Port (or Internal Port): this is the port opening on LAN Host, so the router will forward the requests to.

With our simple server is listening on port 8000, we set the LAN Host Port to 8000 for now. Apply the configuration and go to open port check or open your domain URL directly in browser. You should now see friendly “Hello World!” message.

Note
You should only enable port forwarding on 80 and 443 instead of a wider range.

Now with port forwarding worked as expected. I can switch the forwarding rule to Jellyfin server.

Update our router IP dynamically Link to heading

The public IP on our router is not permanent. It can be changed any time if we reboot router or by the ISP. Now on your computer, run this command if you use Linux or Mac

noip-duc -g all.ddnskey.com -u <DDNSKeyUsername> -p <DDNSKeyPassword>

You can also create a service to run this in background. In my case, I use this docker container as a service.
