I Learn Java

April 7, 2026 2-minute read
In the list of my most hated programming languages, Java is definitely top 1.

My first impression about it was that to run a Hello world, you need to write a lot of code. Even other OOP languages like C++ don’t require that much.

At the time, I was thinking: Yo, these people are really into OOP. But why not just OOP everything and keep the main() function alone? The namespace is weird, why does it take up the whole line with the domains? Most lines of code are just variable name bloat, taking up the whole line and continuing on the next line.

Info
Both Java and C# fixed the main() function issue now, which mean I was right.

Haha, that first impression made me hate Java and C# so bad. While that sounds like kid behavior, I have to admit it saved me from the world of OOP, SOLID, and Clean code. Another time I joined a project with different tech stacks and Java was one of them. I saw meaningless interfaces, only have one implementation, but they had to be there for “cleanness of the code”. The rule was written in the book.

The function was too long, you have to split it into simpler functions. Yeah, of course it would have many LOC because of naming bloat and fucking OOP chaining methods.

I hated it more. I felt disgust.

And then, anytime people wanted me to work on a scope involving Java, I refused immediately.

A few years later, I still hate Java, but in some situations, I had to do integration between micro services, fix some little bugs. I got exposed to more Java code along the way. I tried to get things done and get the hell out as soon as possible.

But today, I am going to jump into this muddy thing that I have hated for so long. Why? For money, of course, LOL. But more importantly, I am ready for a new challenge. And Java is so well-suited for my back end focus roadmap. It will help me expand my back end knowledge and solve problems I have never faced before with other languages. You can’t defeat a warrior that survived so many battles. I think I will stick with Java for at least 2-3 years and see what I would take out of it.
